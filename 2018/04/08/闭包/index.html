2. 关于this对象

this对象是在运行时基于函数的执行环境绑定的：在全局环境中，this等于window，而当函数被作为某个对象的方法调用时，this等于那个对象。不过，匿名函数的执行环境具有全局性，因此其this对象通常指向window。但由于编写闭包的方式不同，这点可能会比较不明显。

当使用前者的构造函数创建出来的实例，私有变量和私有函数都是自己私有的，不与其他实例共享，因此其中一个实例调用特权方法改变了私有变量和私有函数，并不会导致其他实例的私有变量和私有函数发生改变。而后者的匿名函数私有变量和私有函数都是共享的，使用后者的构造函数创建出来的实例，它们的方法都可以访问到这些私有变量和私有函数，如果其中一个实例调用特权方法改变了私有变量和私有函数，这些私有变量和私有函数的变化也会在其他实例中反映出来，于是在这种模式下，私有变量会变成一个静态的、由所有实例共享的属性。

简言之，如果必须创建一个对象并以某些数据对其进行初始化，同时还要公开一些能够访问这些私有数据的方法，那么就可以使用模块模式。以这种模式创建的每个单例都是 Object 的实例，因为最终要通过一个对象字面量来表示它。然而，单例通常都是作为全局对象存在，不会将它传递给一个函数。因此，也就没有什么必要使用 instanceof 操作符来检查其对象类型。

3.2.3. 增强的模块模式

增强的模块模式，即在返回对象之前加入对其增强的代码。这种增强的模块模式适合那些单例必须是某种类型的实例，同时还必须添加某些属性和（或）方法对其加以增强的情况。（详情查看《JavaScript高级程序设计》P191）
